# PakPass: آف لائن گاڑی رجسٹریشن اور کیو آر تصدیقی کوڈ ایپ

پاک پاس ایک 100% آف لائن، انتہائی محفوظ اور جدید اینڈرائیڈ موبائل ایپلی کیشن ہے جو گاڑیوں کی رجسٹریشن اور فوری سیکیورٹی تصدیق کے لیے تیار کی گئی ہے۔ یہ ایپ بغیر انٹرنیٹ کنکشن کے کام کرتی ہے اور تمام ڈیٹا مقامی طور پر اسٹور کرتی ہے۔

## اہم خصوصیات (Features)

1. **مکمل آف لائن آپریشن (100% Offline)**
   - یہ ایپ مکمل سیکیورٹی کے ساتھ آف لائن چلتی ہے۔ تمام ڈیٹا سیکیورٹی کارڈز اور اسکین کوڈز کو غیر فعال آلات پر تصدیق کے لیے استعمال کیا جا سکتا ہے۔
2. **سیکیورٹی اور ہارڈویئر لاک (Hardware-backed Integrity)**
   - پہلی بار ایپ چلانے پر صارف سے 6 ہندسوں کا ماسٹر پن لیا جاتا ہے جس سے **PBKDF2** کے ذریعے AES-256 کی تلاش (Derivation) کی جاتی ہے۔
   - یہ حاصل شدہ خفیہ چابی اینڈرائیڈ کے ہارڈویئر پر مبنی **KeyStore** کے ذریعے محفوظ کی جاتی ہے۔
3. **مضبوط انکرپشن (AES-256-CBC Encryption)**
   - تمام پاسز کی معلومات (نام، شناختی کارڈ، گاڑی نمبر، اور میعاد) کو مقامی چابی کے ساتھ انکرپٹ کر کے بیس 64 (Base64) بائنری مواد میں بدلا جاتا ہے۔
4. **بلیوٹوتھ تھرمل پرنٹر انٹیگریشن (Bluetooth Printer)**
   - ایڈمن پینل سے براہ راست پرنٹ کا بٹن دبا کر کسی بھی 58mm بلیوٹوتھ تھرمل پرنٹر پر گاڑی کا پاس اور تفصیلات پرنٹ کی جا سکتی ہیں۔
5. **آف لائن ڈیٹا بیک اپ اور ری سٹور (Backup & Restore)**
   - سمارٹ فائل سسٹم کے زریعے آپ پورے ڈیٹا بیس کا JSON فارمیٹ میں بیک اپ فائل بنا کر ڈاؤن لوڈ فولڈر میں محفوظ کر سکتے ہیں اور با آسانی امپورٹ کر سکتے ہیں۔

## ڈیٹا بیس کا خاکہ (Database Tables)

### ۱. یوزرز ٹیبل (`users`):
* `id`: خودکار نمبر (Primary Key)
* `username`: صارف کا نام (جیسے admin یا scanner1)
* `pin_hash`: صارف کے پن کا SHA-256 ہیش فورس
* `role`: کردار (ماڈریٹر ایڈمن یا بارکوڈ اسکینر)
* `created_at`: اکاؤنٹ بننے کی تاریخ

### ۲. وہیکل پاسز ٹیبل (`vehicle_passes`):
* `id`: پاس فارم رجسٹریشن (Primary Key)
* `unique_id`: آفاقی شناختی نمبر (UUID v4)
* `owner_name`: مالک کا نام
* `cnic`: شناختی کارڈ نمبر (XXXXX-XXXXXXX-X)
* `vehicle_no`: رجسٹریشن نمبر
* `phone`: فون نمبر (اختیاری)
* `expiry_date`: میعاد ختم ہونے کی تاریخ (ISO 8601)
* `created_at`: پاس بنانے کا درست وقت
* `is_revoked`: منسوخ حالت (0 = فعال، 1 = منسوخ شدہ)

## کوڈ کی تنصیب اور سیٹ اپ (Technical Setup Instructions)

### ضرورتیں (Prerequisites)
- **منزل ایپ پلیٹ فارم (Platform):** Kotlin / Jetpack Compose
- **کم از کم API لیول (Minimum SDK):** API 24 (Android 7.0) اور اس سے زیادہ
- **کیمرہ (Camera):** اسکینر تلاشی پورٹل کے لیے پیچھے والا کیمرہ لازمی ہے۔

### چلانے کا طریقہ کار (How to Run)
1. **پہلی بار چلانے پر (First Run Screen):**
   - ایپ کھولتے ہی آپ سے "ماسٹر ایڈمن پن سیٹ کریں (6 ہندسے)" کا مطالبہ کیا جائے گا۔ پن کو دو بار درج کر کے کنفرم کریں۔
   - اس سے محفوظ ترین سیکیورٹی نظام فعال ہو جائے گا۔
2. **لاگ ان کریں (Login Screen):**
   - ہوم اسکرین پر اپنے رجسٹرڈ صارف نام اور 6 ہندسوں کا پن کوڈ لگا کر لاگ ان کریں۔
3. **پرنٹ کریں (Thermal Printer):**
   - بلیوٹوتھ پرنٹر کو آن کریں، اور اپنے فون کی ریگولر سیٹنگز سے جوڑیں (Pair)۔
   - پاس بنانے پر "پرنٹ" بٹن دبا کر کلاسک پرنٹر منتخب کریں تو فوری طور پر پرنٹ ہو جائے گا۔

## آف لائن سیکیورٹی اور وفاداری (Offline Security Model)
تصدیقی عمل میں سیکیورٹی گارڈز بارکوڈ ریڈر کھولتے ہیں۔ جب کیو آر کوڈ اسکین ہوتا ہے، تو کیمرہ انکرپٹڈ بیس 64 ڈیٹا کو ڈیکوڈ کر کے آف لائن ڈکرپشن انجام دیتا ہے۔
اگر کوئی سیکیورٹی چابی کو چھیڑے گا یا خود سے جعلی پاس بنانے کی کوشش کرے گا، تو لوکل ڈی کرپشن ناکام ہو جائے گی اور سیکیورٹی سکرین فوراً سرخ ہو کر **"جعلی پاس - INVALID"** کا الارم دکھائے گی۔
اگر کارڈ کو منسوخ (Revoke) کر دیا گیا ہو، تو مقامی ڈیٹا بیس سے میچ کر کے گارڈ کو **"منسوخ شدہ پاس"** کا انتباہ دیا جائے گا۔
